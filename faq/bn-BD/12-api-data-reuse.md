## আপলোড করা ছবির জন্য কোন আকারটি ভালো হবে সে সম্পর্কে ডকুমেন্টেশনের কোথাও কি কোন সুপারিশ আছে?

নেটওয়ার্ক ধীর বা ব্যয়বহুল কিনা তা দেশগুলির উপর নির্ভর করতে পারে। ৫০০০ পিক্সেলের বেশি ওজন বা উচ্চতা সম্ভবত খুব একটা কার্যকর নয়। আর যদি আপনি কোনওভাবে বুঝতে পারেন যে নেটওয়ার্ক ধীর, তাহলে ২০০০ পিক্সেলের ছবিও দুর্দান্ত হবে (ছবি না থাকার চেয়ে অবশ্যই ভালো!)

## বারকোড ছাড়া খাবারের কী হবে?

ওপেন ফুড ফ্যাক্টস-এ কেবল প্যাকেটজাত খাবার সম্পর্কে তথ্য রয়েছে। উৎপাদনের গড় মূল্য (যেমন, টমেটো বা কলা) এবং অন্যান্য খাদ্যপণ্যের জন্য, আপনি এর পরিবর্তে সরকারী জাতীয় পুষ্টি ডাটাবেসগুলির একটি ব্যবহার করতে পারেন।

**বিঃদ্রঃ:** নীচের তালিকায় কিছু গুরুত্বপূর্ণ জাতীয় খাদ্য ডাটাবেস রয়েছে। যদি আপনার মনে হয় তালিকায় অন্য কোনও ডাটাবেস অন্তর্ভুক্ত করা উচিত, তাহলে অনুগ্রহ করে আমাদের সাথে যোগাযোগ করুন: [https://world.openfoodfacts.org/contact](https://world.openfoodfacts.org/contact)

## আমি কি API ব্যবহার করে একটি সুনির্দিষ্ট পণ্যের নাম অনুসন্ধান করতে পারি?

দুর্ভাগ্যবশত API-এর মাধ্যমে কেবল পণ্যের নামে এবং সঠিকভাবে অনুসন্ধান করা এখনও সম্ভব হয়নি।

তবে ক্যাটাগরি অনুযায়ী ফিল্টার ব্যবহার করলে আপনার অনুসন্ধান আরও সুনির্দিষ্ট হতে পারে।

---

## আমি কিভাবে আমার প্রকল্পগুলির জন্য তথ্য অ্যাক্সেস/সংগ্রহ করতে পারি?

ওপেন ফুড ফ্যাক্টসের মূল পৃষ্ঠায়, স্ক্রিনের উপরের বাম কোণে, একটি স্ক্রোলিং মেনু রয়েছে। এর নীচে, আপনি "উন্নত অনুসন্ধান" বিকল্পটি পাবেন, যার উপর আপনি ক্লিক করতে পারেন। এরপর আপনার প্রকল্পের জন্য কোন মানদণ্ড সবচেয়ে প্রাসঙ্গিক তা নির্ধারণ করা আপনার উপর নির্ভর করে। একবার বেছে নিলে, আপনি পৃষ্ঠার নীচে স্ক্রোল করে এবং "ফলাফল ডাউনলোড করুন" এ ক্লিক করে প্রাপ্ত ফলাফলগুলি ডাউনলোড করতে সক্ষম হবেন।

আপনি পরামর্শও করতে পারেন:

- আমাদের API ডকুমেন্টেশন:[ https://openfoodfacts.github.io/api-documentation/](https://openfoodfacts.github.io/api-documentation/)
- ওপেন ফুড ফ্যাক্টস ব্যবহারের শর্তাবলী:[ ](https://world.openfoodfacts.org/terms-of-use)[https://world.openfoodfacts.org/terms-of-use](https://world.openfoodfacts.org/terms-of-use)
- আমাদের তথ্যের উপর:[ https://world.openfoodfacts.org/data](https://world.openfoodfacts.org/data)

## API ব্যবহারের জন্য কি কোন শর্ত আছে?

API ব্যবহার সম্পর্কে সমস্ত ডকুমেন্টেশন [API ডকুমেন্টেশন পৃষ্ঠা](https://openfoodfacts.github.io/openfoodfacts-server/api/) এ পাওয়া যাবে, তবে এখানে একটি সংক্ষিপ্ত সারাংশ দেওয়া হল:

- ওপেন ফুড ফ্যাক্টস ডাটাবেসটি ওপেন ডাটাবেস লাইসেন্স (ODbL) এর অধীনে ওপেন ডেটা হিসাবে উপলব্ধ, আইনি বিবরণের জন্য [https://world.openfoodfacts.org/terms-of-use](https://world.openfoodfacts.org/terms-of-use) দেখুন। দুটি শর্ত হল অ্যাট্রিবিউশন এবং শেয়ার-অ্যালাইক। যদি আপনি ওপেন ফুড ফ্যাক্টস থেকে প্রাপ্ত তথ্য অন্যান্য ডাটাবেসের সাথে একত্রিত করেন, তাহলে ODbL-এর জন্য প্রয়োজন যে ফলস্বরূপ ডাটাবেসটিও ওপেন ডেটা হিসেবে প্রকাশ করা উচিত। এর অর্থ হল আপনি কেবলমাত্র সেই উৎসগুলির সাথে ডেটা একত্রিত করতে পারেন যা এই ধরনের পুনর্বণ্টনের অনুমতি দেবে।

- আপনার অ্যাপ শনাক্ত করার জন্য API কল করার সময় আপনাকে অবশ্যই **সর্বদা** একটি কাস্টম ইউজার-এজেন্ট ব্যবহার করতে হবে।

- প্রতিটি API এন্ডপয়েন্টের জন্য রেট-সীমা প্রয়োগ করা হয়।

## আমি কিভাবে ঐতিহাসিক তথ্য অ্যাক্সেস করতে পারি?

বর্তমানে, আমরা ঐতিহাসিক ডেটা ডাম্প (JSONL, MongoDB, CSV) অফার করি না।

তবে, পৃথক পণ্যের জন্য, API ব্যবহার করে বা পণ্য পৃষ্ঠায় সংশোধন ব্যবহার করে পণ্যের ডেটার পূর্ববর্তী সংস্করণগুলি অ্যাক্সেস করা সম্ভব।

প্রতিবার যখন কোনও পণ্য আপডেট করা হয়, তখন একটি নতুন সংশোধন (১ থেকে শুরু করে বর্ধিত সংখ্যা) তৈরি করা হয়।

উদাহরণস্বরূপ, এই পণ্যের প্রথম সংস্করণ (=প্রথম পণ্য সংস্করণ) পেতে, ব্যবহার করুন

[https://world.openfoodfacts.org/product/7623186089763/joghurt-baumnuss-migros?rev=1](https://world.openfoodfacts.org/product/7623186089763/joghurt-baumnuss-migros?rev=1).

একইভাবে, rev প্যারামিটারটি API এর সাথে ব্যবহার করা যেতে পারে:

[https://world.openfoodfacts.org/api/v2/product/7623186089763?rev=1](https://world.openfoodfacts.org/api/v2/product/7623186089763?rev=1)
