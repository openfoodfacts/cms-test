## Փաստաթղթերում կա՞ն առաջարկություններ այն մասին, թե վերբեռնված լուսանկարների չափը որ չափսն է լավ։

Դա կարող է կախված լինել երկրներից, եթե ցանցը դանդաղ է կամ թանկ։ 5000 պիքսելից բարձր քաշով կամ բարձրությամբ ամեն ինչ, հավանաբար, այդքան էլ օգտակար չէ։ և եթե դուք կարողանաք ինչ-որ կերպ հայտնաբերել, որ ցանցը դանդաղ է, ապա նույնիսկ 2000 պիքսելանոց պատկերը հիանալի կլինի (իհարկե, ավելի լավ է, քան պատկեր չունենալը):

## Իսկ ի՞նչ կասեք շտրիխ կոդեր չունեցող սննդի մասին։

Բաց սննդի փաստերը պարունակում են միայն փաթեթավորված սննդի մասին տեղեկատվություն։ Մթերքների (օրինակ՝ լոլիկի կամ բանանի) և այլ սննդամթերքի միջին արժեքների համար կարող եք օգտագործել պաշտոնական ազգային սննդային տվյալների բազաներից մեկը։

**Նշում.** Ստորև բերված ցանկը պարունակում է սննդի ամենակարևոր ազգային տվյալների բազաներից մի քանիսը։ Եթե կարծում եք, որ ցանկում պետք է ներառվի որևէ այլ տվյալների բազա, խնդրում ենք կապվել մեզ հետ հետևյալ հասցեով՝ [https://world.openfoodfacts.org/contact](https://world.openfoodfacts.org/contact)

## Կարո՞ղ եմ API-ի միջոցով որոնել ճշգրիտ ապրանքի անվանումը։

Դժբախտաբար, դեռևս հնարավոր չէ հեշտությամբ և ճշգրիտ որոնել միայն ապրանքի անվանմամբ API-ի միջոցով։

Կատեգորիայի վրա ֆիլտրի օգտագործումը կարող է օգնել ձեզ ավելի ճշգրիտ դարձնել ձեր որոնումը։

---

## Ինչպե՞ս կարող եմ մուտք գործել/հավաքել տվյալներ իմ նախագծերի համար։

«Բաց սննդի փաստեր»-ի գլխավոր էջում, էկրանի վերին ձախ անկյունում, կա գլորվող ցանկ։ Ներքևի մասում կգտնեք «Ընդլայնված որոնում» տարբերակը, որի վրա կարող եք սեղմել։ Այնուհետև ձեզանից է կախված որոշել, թե որ չափանիշներն են առավել համապատասխան ձեր նախագծին(ներին): Ընտրելուց հետո դուք կկարողանաք ներբեռնել ստացված արդյունքները՝ էջի ներքևի մասում գլորելով ներքև և սեղմելով «Ներբեռնել արդյունքները» կոճակը։

Կարող եք նաև խորհրդակցել՝

- Մեր API փաստաթղթերը՝ [ https://openfoodfacts.github.io/api-documentation/](https://openfoodfacts.github.io/api-documentation/)
- «Բաց սննդի փաստեր»-ի օգտագործման պայմաններ՝[ ](https://world.openfoodfacts.org/terms-of-use)[https://world.openfoodfacts.org/terms-of-use](https://world.openfoodfacts.org/terms-of-use)
- Մեր տվյալների հիման վրա՝ [ https://world.openfoodfacts.org/data](https://world.openfoodfacts.org/data)

## Կա՞ն պայմաններ API-ն օգտագործելու համար։

API-ի օգտագործման վերաբերյալ բոլոր փաստաթղթերը կարելի է գտնել [API փաստաթղթերի էջում](https://openfoodfacts.github.io/openfoodfacts-server/api/), բայց ահա դրանց համառոտ ամփոփումը.

- Բաց սննդի փաստերի տվյալների բազան հասանելի է որպես բաց տվյալներ Բաց տվյալների բազայի լիցենզիայով (ODbL), իրավական մանրամասների համար տե՛ս [https://world.openfoodfacts.org/terms-of-use](https://world.openfoodfacts.org/terms-of-use): Երկու պայմաններն են վերագրումը և համարժեք կիսվելը։ Եթե դուք համատեղում եք Open Food Facts-ի տվյալները այլ տվյալների բազաների հետ, ապա ODbL-ը պահանջում է, որ արդյունքում ստացված տվյալների բազան նույնպես հրապարակվի որպես բաց տվյալներ։ Դա նաև նշանակում է, որ դուք կարող եք տվյալները համատեղել միայն այն աղբյուրների հետ, որոնք թույլ կտան նման վերաբաշխում։

- Դուք պետք է **միշտ** օգտագործեք հատուկ User-Agent՝ ձեր հավելվածը նույնականացնելու համար API կանչեր կատարելիս։

- Յուրաքանչյուր API վերջնակետի համար կիրառվում են արագության սահմանափակումներ։

## Ինչպե՞ս կարող եմ մուտք գործել պատմական տվյալներին։

Ներկայումս մենք չենք առաջարկում պատմական տվյալների կրկնօրինակում (JSONL, MongoDB, CSV):

Այնուամենայնիվ, առանձին ապրանքների համար հնարավոր է մուտք գործել ապրանքի տվյալների նախորդ տարբերակներին՝ օգտագործելով API-ը կամ ապրանքի էջում՝ օգտագործելով վերանայումները։

Ամեն անգամ, երբ ապրանքը թարմացվում է, ստեղծվում է նոր վերանայում (աճող թվանշան՝ սկսած 1-ից):

Օրինակ, այս ապրանքի առաջին տարբերակը (=առաջին ապրանքի տարբերակը) ստանալու համար օգտագործեք
