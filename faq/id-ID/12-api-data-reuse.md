## Apakah ada rekomendasi di dalam dokumentasi mengenai ukuran yang ideal untuk foto yang diunggah?

Hal itu mungkin bergantung pada negara, apakah jaringan internet lambat atau mahal. Ukuran atau lebar apa pun yang melebihi 5000 piksel mungkin tidak terlalu berguna. dan jika Anda dapat mendeteksi bahwa jaringan lambat, maka gambar 2000 piksel pun akan sangat membantu (tentu lebih baik daripada tidak memiliki gambar sama sekali!).

## Bagaimana cara saya mengakses/mengumpulkan data untuk proyek saya?

Di halaman utama Open Food Facts, di pojok kiri atas layar, terdapat menu yang dapat digulir. Di bagian bawahnya, Anda akan menemukan opsi "pencarian lanjutan", yang dapat Anda klik. Selanjutnya, terserah Anda untuk menentukan kriteria mana yang paling relevan dengan proyek Anda. Setelah dipilih, Anda dapat mengunduh hasil yang diperoleh dengan menggulir ke bawah halaman dan mengklik "Unduh hasil".

Anda juga dapat berkonsultasi dengan:

- Dokumentasi API kami:[ https://openfoodfacts.github.io/api-documentation/](https://openfoodfacts.github.io/api-documentation/)
- Ketentuan Penggunaan Open Food Facts:[ ](https://world.openfoodfacts.org/terms-of-use)[https://world.openfoodfacts.org/terms-of-use](https://world.openfoodfacts.org/terms-of-use)
- Berdasarkan data kami:[ https://world.openfoodfacts.org/data](https://world.openfoodfacts.org/data)

## Apakah ada persyaratan untuk menggunakan API?

Semua dokumentasi tentang penggunaan API dapat ditemukan di [halaman dokumentasi API](https://openfoodfacts.github.io/openfoodfacts-server/api/), tetapi berikut ringkasan singkatnya:

- Basis data Open Food Facts tersedia sebagai data terbuka di bawah Lisensi Basis Data Terbuka (ODbL), lihat [https://world.openfoodfacts.org/terms-of-use](https://world.openfoodfacts.org/terms-of-use) untuk detail hukumnya. Kedua kondisi tersebut adalah atribusi dan berbagi secara setara. Jika Anda menggabungkan data dari Open Food Facts dengan basis data lain, maka ODbL mensyaratkan bahwa basis data yang dihasilkan juga harus dirilis sebagai data terbuka. Ini juga berarti bahwa Anda hanya dapat menggabungkan data dengan sumber yang mengizinkan pendistribusian ulang tersebut.

- Anda harus **selalu** menggunakan User-Agent khusus saat melakukan panggilan API untuk mengidentifikasi aplikasi Anda.

- Pembatasan laju permintaan diberlakukan untuk setiap titik akhir API.

## Bagaimana cara saya mengakses data historis?

Saat ini, kami tidak menyediakan data historis (JSONL, MongoDB, CSV).

Namun, untuk produk individual, dimungkinkan untuk mengakses versi data produk sebelumnya menggunakan API atau di halaman produk menggunakan revisi.

Setiap kali suatu produk diperbarui, revisi baru (angka yang meningkat mulai dari 1) akan dibuat.

Misalnya, untuk mendapatkan revisi pertama (=versi produk pertama) dari produk ini, gunakan

[https://world.openfoodfacts.org/product/7623186089763/joghurt-baumnuss-migros?rev=1](https://world.openfoodfacts.org/product/7623186089763/joghurt-baumnuss-migros?rev=1).

Demikian pula, parameter rev dapat digunakan dengan API:

[https://world.openfoodfacts.org/api/v2/product/7623186089763?rev=1](https://world.openfoodfacts.org/api/v2/product/7623186089763?rev=1)
