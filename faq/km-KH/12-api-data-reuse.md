## តើមានការណែនាំនៅកន្លែងណាមួយនៅក្នុងឯកសារអំពីទំហំដែលល្អសម្រាប់រូបថតដែលបានផ្ទុកឡើងដែរឬទេ?

នោះ​អាច​អាស្រ័យ​លើ​ប្រទេស​នានា ថាតើ​បណ្ដាញ​យឺត ឬ​ថ្លៃ​ឬ​អត់។ អ្វីដែលលើសពី 5000 ភីកសែល ទាំងទម្ងន់ ឬកម្ពស់ ប្រហែលជាមិនមានប្រយោជន៍ខ្លាំងទេ។ ហើយប្រសិនបើអ្នកអាចរកឃើញថាបណ្តាញយឺត នោះសូម្បីតែរូបភាព 2000 ភីកសែលក៏ល្អដែរ (ពិតជាប្រសើរជាងការមិនមានរូបភាព!)

## ចុះ​អាហារ​ដែល​គ្មាន​បាកូដ​វិញ?

Open Food Facts មានតែព័ត៌មានអំពីអាហារដែលបានវេចខ្ចប់ប៉ុណ្ណោះ។ ចំពោះតម្លៃជាមធ្យមនៃផលិតផល (ឧទាហរណ៍ ប៉េងប៉ោះ ឬចេក) និងផលិតផលអាហារផ្សេងទៀត អ្នកអាចប្រើមូលដ្ឋានទិន្នន័យអាហារូបត្ថម្ភជាតិផ្លូវការមួយជំនួសវិញ។

**ចំណាំ៖** បញ្ជីខាងក្រោមមានមូលដ្ឋានទិន្នន័យអាហារជាតិសំខាន់ៗមួយចំនួន។ ប្រសិនបើអ្នកគិតថាមូលដ្ឋានទិន្នន័យផ្សេងទៀតគួរតែត្រូវបានបញ្ចូលក្នុងបញ្ជី សូមទាក់ទងមកយើងខ្ញុំតាមរយៈ៖ [https://world.openfoodfacts.org/contact](https://world.openfoodfacts.org/contact)

## តើខ្ញុំអាចស្វែងរកឈ្មោះផលិតផលជាក់លាក់មួយដោយប្រើ API បានទេ?

ជាអកុសល វាមិនទាន់អាចស្វែងរកបានយ៉ាងងាយស្រួលលើឈ្មោះផលិតផលតែប៉ុណ្ណោះ និងយ៉ាងច្បាស់លាស់តាមរយៈ API នៅឡើយទេ។

ទោះជាយ៉ាងណាក៏ដោយ ការប្រើប្រាស់តម្រងលើប្រភេទអាចជួយអ្នកធ្វើឱ្យការស្វែងរករបស់អ្នកកាន់តែច្បាស់លាស់។

## តើខ្ញុំអាចចូលមើល/ប្រមូលទិន្នន័យសម្រាប់គម្រោងរបស់ខ្ញុំដោយរបៀបណា?

នៅលើទំព័រមេរបស់ Open Food Facts នៅជ្រុងខាងឆ្វេងខាងលើនៃអេក្រង់ មានម៉ឺនុយរំកិលមួយ។ នៅផ្នែកខាងក្រោមរបស់វា អ្នកនឹងឃើញជម្រើស "ការស្វែងរកកម្រិតខ្ពស់" ដែលអ្នកអាចចុចលើវាបាន។ បន្ទាប់មក វាអាស្រ័យលើអ្នកក្នុងការកំណត់ថាលក្ខណៈវិនិច្ឆ័យណាដែលពាក់ព័ន្ធបំផុតចំពោះគម្រោងរបស់អ្នក។ នៅពេលជ្រើសរើសរួច អ្នកនឹងអាចទាញយកលទ្ធផលដែលទទួលបានដោយរំកិលចុះក្រោមនៅផ្នែកខាងក្រោមនៃទំព័រ ហើយចុចលើ "ទាញយកលទ្ធផល"។

អ្នកក៏អាចពិគ្រោះជាមួយ៖

- ឯកសារ API របស់យើង៖[ https://openfoodfacts.github.io/api-documentation/](https://openfoodfacts.github.io/api-documentation/)
- លក្ខខណ្ឌនៃការប្រើប្រាស់របស់ Open Food Facts៖[ ](https://world.openfoodfacts.org/terms-of-use)[https://world.openfoodfacts.org/terms-of-use](https://world.openfoodfacts.org/terms-of-use)
- លើទិន្នន័យរបស់យើង៖[ https://world.openfoodfacts.org/data](https://world.openfoodfacts.org/data)

## តើមានលក្ខខណ្ឌដើម្បីប្រើប្រាស់ API ដែរឬទេ?

ឯកសារទាំងអស់អំពីការប្រើប្រាស់ API អាចរកបាននៅលើ [ទំព័រឯកសារ API](https://openfoodfacts.github.io/openfoodfacts-server/api/) ប៉ុន្តែនេះគឺជាសេចក្តីសង្ខេបរហ័សមួយ៖

- មូលដ្ឋានទិន្នន័យ Open Food Facts អាចរកបានជាទិន្នន័យបើកចំហក្រោមអាជ្ញាប័ណ្ណ Open Database (ODbL) សូមមើល [https://world.openfoodfacts.org/terms-of-use](https://world.openfoodfacts.org/terms-of-use) សម្រាប់ព័ត៌មានលម្អិតផ្នែកច្បាប់។ លក្ខខណ្ឌទាំងពីរគឺ ការបញ្ជាក់កម្មសិទ្ធិ និងការចែករំលែកដូចគ្នា។ ប្រសិនបើអ្នកផ្សំទិន្នន័យពី Open Food Facts ជាមួយមូលដ្ឋានទិន្នន័យផ្សេងទៀត នោះ ODbL តម្រូវឱ្យមូលដ្ឋានទិន្នន័យលទ្ធផលត្រូវតែចេញផ្សាយជាទិន្នន័យបើកចំហផងដែរ។ វាក៏មានន័យថា អ្នកអាចផ្សំទិន្នន័យបានតែជាមួយប្រភពដែលនឹងអនុញ្ញាតឱ្យមានការចែកចាយឡើងវិញបែបនេះ។

- អ្នកត្រូវតែ **តែងតែ** ប្រើ User-Agent ផ្ទាល់ខ្លួន នៅពេលអនុវត្តការហៅ API ដើម្បីកំណត់អត្តសញ្ញាណកម្មវិធីរបស់អ្នក។

- ដែនកំណត់អត្រាត្រូវបានអនុវត្តសម្រាប់ចំណុចបញ្ចប់ API នីមួយៗ។

## តើខ្ញុំអាចចូលប្រើទិន្នន័យប្រវត្តិសាស្ត្រដោយរបៀបណា?

បច្ចុប្បន្ននេះ យើងមិនផ្តល់ជូនការផ្ទេរទិន្នន័យប្រវត្តិសាស្ត្រ (JSONL, MongoDB, CSV) ទេ។

ទោះយ៉ាងណាក៏ដោយ សម្រាប់ផលិតផលនីមួយៗ វាអាចចូលប្រើទិន្នន័យផលិតផលកំណែមុនៗដោយប្រើ API ឬនៅលើទំព័រផលិតផលដោយប្រើការកែប្រែ។

រាល់ពេលដែលផលិតផលត្រូវបានធ្វើបច្ចុប្បន្នភាព ការកែប្រែថ្មី (បង្កើនខ្ទង់ចាប់ផ្តើមពី 1) ត្រូវបានបង្កើតឡើង។

ឧទាហរណ៍ ដើម្បីទទួលបានការកែប្រែដំបូង (=កំណែផលិតផលដំបូង) នៃផលិតផលនេះ សូមប្រើ

[https://world.openfoodfacts.org/product/7623186089763/joghurt-baumnuss-migros?rev=1](https://world.openfoodfacts.org/product/7623186089763/joghurt-baumnuss-migros?rev=1)។
