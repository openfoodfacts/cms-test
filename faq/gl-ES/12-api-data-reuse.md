## Hai recomendacións nalgún lugar da documentación sobre cal sería un bo tamaño para as fotos subidas?

Iso pode depender dos países, se a rede é lenta ou cara. Calquera cousa que teña máis de 5000 píxeles en peso ou altura probablemente non sexa moi útil. e se dalgún xeito podes detectar que a rede é lenta, entón mesmo unha imaxe de 2000 píxeles sería xenial (sen dúbida mellor que non ter imaxe!).

## E que pasa coa comida sen código de barras?

Open Food Facts só contén información sobre alimentos envasados. Para os valores medios de produtos agrícolas (por exemplo, tomates ou plátanos) e outros produtos alimenticios, podes usar unha das bases de datos nacionais oficiais de nutrición.

**Nota:** A lista que aparece a continuación contén algunhas das bases de datos nacionais de alimentos máis importantes. Se cres que se debería incluír algunha outra base de datos na lista, ponte en contacto connosco en: [https://world.openfoodfacts.org/contact](https://world.openfoodfacts.org/contact)

## Podo buscar un nome preciso de produto coa API?

Desafortunadamente, aínda non é posible buscar facilmente só polo nome do produto e de forma precisa a través da API.

Non obstante, usar un filtro por categoría pode axudarche a que a túa busca sexa máis precisa.

## Como podo acceder/recopilar datos para os meus proxectos?

Na páxina principal de Open Food Facts, na esquina superior esquerda da pantalla, hai un menú desplegable. Na parte inferior atoparás a opción de "busca avanzada", na que podes premer. Despois, depende de ti determinar cales son os criterios máis relevantes para o teu(s) proxecto(s). Unha vez escollido, poderás descargar os resultados obtidos desprazándote cara abaixo na parte inferior da páxina e facendo clic en "Descargar resultados".

Tamén podes consultar:

- A nosa documentación da API:[ https://openfoodfacts.github.io/api-documentation/](https://openfoodfacts.github.io/api-documentation/)
- Condicións de uso de Open Food Facts:[ ](https://world.openfoodfacts.org/terms-of-use)[https://world.openfoodfacts.org/terms-of-use](https://world.openfoodfacts.org/terms-of-use)
- Nos nosos datos:[ https://world.openfoodfacts.org/data](https://world.openfoodfacts.org/data)

## Hai condicións para usar a API?

Toda a documentación sobre o uso da API pódese atopar na [páxina de documentación da API](https://openfoodfacts.github.io/openfoodfacts-server/api/), pero aquí tes un breve resumo:

- A base de datos Open Food Facts está dispoñible como datos abertos baixo a licenza de base de datos aberta (ODbL); consulte [[https://world.openfoodfacts.org/terms-of-use](https://world.openfoodfacts.org/terms-of-use) para obter información legal. As dúas condicións son atribución e compartición igual. Se combinas datos de Open Food Facts con outras bases de datos, a ODbL require que a base de datos resultante tamén se publique como datos abertos. Tamén significa que só podes combinar os datos con fontes que permitan tal redistribución.

- Debes **sempre** usar un axente de usuario personalizado ao realizar chamadas á API para identificar a túa aplicación.

- Os límites de velocidade aplícanse para cada punto final da API.

## Como podo acceder a datos históricos?

Actualmente, non ofrecemos volcado de datos históricos (JSONL, MongoDB, CSV).

Non obstante, para produtos individuais, é posible acceder a versións anteriores dos datos do produto mediante a API ou na páxina do produto mediante revisións.

Cada vez que se actualiza un produto, créase unha nova revisión (díxito crecente a partir de 1).

Por exemplo, para obter a primeira revisión (=primeira versión do produto) deste produto, use

[https://world.openfoodfacts.org/product/7623186089763/joghurt-baumnuss-migros?rev=1](https://world.openfoodfacts.org/product/7623186089763/jogurt-baumnuss-migros?rev=1).

Do mesmo xeito, o parámetro rev pódese usar coa API:

[https://world.openfoodfacts.org/api/v2/product/7623186089763?rev=1](https://world.openfoodfacts.org/api/v2/product/7623186089763?rev=1)
