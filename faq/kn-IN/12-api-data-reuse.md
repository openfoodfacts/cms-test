## ಅಪ್‌ಲೋಡ್ ಮಾಡಿದ ಫೋಟೋಗಳಿಗೆ ಯಾವ ಗಾತ್ರ ಉತ್ತಮವಾಗಿರುತ್ತದೆ ಎಂಬುದರ ಕುರಿತು ದಸ್ತಾವೇಜನ್ನುಗಳಲ್ಲಿ ಎಲ್ಲಿಯಾದರೂ ಶಿಫಾರಸುಗಳಿವೆಯೇ?

ನೆಟ್‌ವರ್ಕ್ ನಿಧಾನವಾಗಿದ್ದರೆ ಅಥವಾ ದುಬಾರಿಯಾಗಿದ್ದರೆ ಅದು ದೇಶಗಳನ್ನು ಅವಲಂಬಿಸಿರಬಹುದು. 5000 ಪಿಕ್ಸೆಲ್‌ಗಳಿಗಿಂತ ಹೆಚ್ಚಿನ ತೂಕ ಅಥವಾ ಎತ್ತರವು ಬಹುಶಃ ಹೆಚ್ಚು ಉಪಯುಕ್ತವಲ್ಲ. ಮತ್ತು ನೀವು ಹೇಗಾದರೂ ನೆಟ್‌ವರ್ಕ್ ನಿಧಾನವಾಗಿದೆ ಎಂದು ಪತ್ತೆಹಚ್ಚಲು ಸಾಧ್ಯವಾದರೆ, 2000 ಪಿಕ್ಸೆಲ್‌ಗಳ ಚಿತ್ರವೂ ಉತ್ತಮವಾಗಿರುತ್ತದೆ (ಚಿತ್ರ ಇಲ್ಲದಿರುವುದಕ್ಕಿಂತ ಖಂಡಿತವಾಗಿಯೂ ಉತ್ತಮ!)

## ಬಾರ್‌ಕೋಡ್‌ಗಳಿಲ್ಲದ ಆಹಾರದ ಬಗ್ಗೆ ಏನು?

ಓಪನ್ ಫುಡ್ ಫ್ಯಾಕ್ಟ್ಸ್ ಪ್ಯಾಕೇಜ್ ಮಾಡಿದ ಆಹಾರದ ಬಗ್ಗೆ ಮಾತ್ರ ಮಾಹಿತಿಯನ್ನು ಒಳಗೊಂಡಿದೆ. ಉತ್ಪನ್ನಗಳು (ಉದಾಹರಣೆಗೆ, ಟೊಮ್ಯಾಟೊ ಅಥವಾ ಬಾಳೆಹಣ್ಣುಗಳು) ಮತ್ತು ಇತರ ಆಹಾರ ಉತ್ಪನ್ನಗಳ ಸರಾಸರಿ ಮೌಲ್ಯಗಳಿಗಾಗಿ, ನೀವು ಅಧಿಕೃತ ರಾಷ್ಟ್ರೀಯ ಪೌಷ್ಟಿಕಾಂಶ ದತ್ತಸಂಚಯಗಳಲ್ಲಿ ಒಂದನ್ನು ಬಳಸಬಹುದು.

**ಗಮನಿಸಿ:** ಕೆಳಗಿನ ಪಟ್ಟಿಯು ಕೆಲವು ಪ್ರಮುಖ ರಾಷ್ಟ್ರೀಯ ಆಹಾರ ದತ್ತಸಂಚಯಗಳನ್ನು ಒಳಗೊಂಡಿದೆ. ಪಟ್ಟಿಯಲ್ಲಿ ಬೇರೆ ಯಾವುದಾದರೂ ಡೇಟಾಬೇಸ್ ಸೇರಿಸಬೇಕೆಂದು ನೀವು ಭಾವಿಸಿದರೆ, ದಯವಿಟ್ಟು ನಮ್ಮನ್ನು ಇಲ್ಲಿ ಸಂಪರ್ಕಿಸಿ: [https://world.openfoodfacts.org/contact](https://world.openfoodfacts.org/contact)

## API ಬಳಸಿ ಉತ್ಪನ್ನದ ನಿಖರವಾದ ಹೆಸರನ್ನು ನಾನು ಹುಡುಕಬಹುದೇ?

ದುರದೃಷ್ಟವಶಾತ್ ಉತ್ಪನ್ನದ ಹೆಸರಿನಲ್ಲಿ ಮಾತ್ರ ಸುಲಭವಾಗಿ ಮತ್ತು ನಿಖರವಾಗಿ API ಮೂಲಕ ಹುಡುಕಲು ಇನ್ನೂ ಸಾಧ್ಯವಾಗಿಲ್ಲ.

ವರ್ಗದಲ್ಲಿ ಫಿಲ್ಟರ್ ಬಳಸುವುದರಿಂದ ನಿಮ್ಮ ಹುಡುಕಾಟವನ್ನು ಹೆಚ್ಚು ನಿಖರವಾಗಿ ಮಾಡಲು ಸಹಾಯವಾಗುತ್ತದೆ.

## ನನ್ನ ಯೋಜನೆಗಳಿಗೆ ಡೇಟಾವನ್ನು ನಾನು ಹೇಗೆ ಪ್ರವೇಶಿಸಬಹುದು/ಸಂಗ್ರಹಿಸಬಹುದು?

ಓಪನ್ ಫುಡ್ ಫ್ಯಾಕ್ಟ್ಸ್‌ನ ಮುಖ್ಯ ಪುಟದಲ್ಲಿ, ಪರದೆಯ ಮೇಲಿನ ಎಡ ಮೂಲೆಯಲ್ಲಿ, ಸ್ಕ್ರೋಲಿಂಗ್ ಮೆನು ಇದೆ. ಅದರ ಕೆಳಭಾಗದಲ್ಲಿ, ನೀವು "ಸುಧಾರಿತ ಹುಡುಕಾಟ" ಆಯ್ಕೆಯನ್ನು ಕಾಣುವಿರಿ, ಅದರ ಮೇಲೆ ನೀವು ಕ್ಲಿಕ್ ಮಾಡಬಹುದು. ನಂತರ ನಿಮ್ಮ ಯೋಜನೆ(ಗಳು)ಗೆ ಯಾವ ಮಾನದಂಡಗಳು ಹೆಚ್ಚು ಪ್ರಸ್ತುತವಾಗಿವೆ ಎಂಬುದನ್ನು ನಿರ್ಧರಿಸುವುದು ನಿಮಗೆ ಬಿಟ್ಟದ್ದು. ಆಯ್ಕೆ ಮಾಡಿದ ನಂತರ, ಪುಟದ ಕೆಳಭಾಗದಲ್ಲಿ ಸ್ಕ್ರೋಲ್ ಮಾಡುವ ಮೂಲಕ ಮತ್ತು "ಫಲಿತಾಂಶಗಳನ್ನು ಡೌನ್‌ಲೋಡ್ ಮಾಡಿ" ಕ್ಲಿಕ್ ಮಾಡುವ ಮೂಲಕ ನೀವು ಪಡೆದ ಫಲಿತಾಂಶಗಳನ್ನು ಡೌನ್‌ಲೋಡ್ ಮಾಡಲು ಸಾಧ್ಯವಾಗುತ್ತದೆ.

ನೀವು ಸಹ ಸಂಪರ್ಕಿಸಬಹುದು:

- ನಮ್ಮ API ದಸ್ತಾವೇಜೀಕರಣ:[ https://openfoodfacts.github.io/api-documentation/](https://openfoodfacts.github.io/api-documentation/)
- ಓಪನ್ ಫುಡ್ ಫ್ಯಾಕ್ಟ್ಸ್ ಬಳಕೆಯ ನಿಯಮಗಳು:[ ](https://world.openfoodfacts.org/terms-of-use)[https://world.openfoodfacts.org/terms-of-use](https://world.openfoodfacts.org/terms-of-use)
- ನಮ್ಮ ಡೇಟಾದಲ್ಲಿ:[ https://world.openfoodfacts.org/data](https://world.openfoodfacts.org/data)

## API ಬಳಸಲು ಯಾವುದೇ ಷರತ್ತುಗಳಿವೆಯೇ?

API ಬಳಕೆಯ ಕುರಿತಾದ ಎಲ್ಲಾ ದಸ್ತಾವೇಜನ್ನು [API ದಸ್ತಾವೇಜೀಕರಣ ಪುಟ](https://openfoodfacts.github.io/openfoodfacts-server/api/ನಲ್ಲಿ ಕಾಣಬಹುದು), ಆದರೆ ಇಲ್ಲಿ ಒಂದು ಸಣ್ಣ ಸಾರಾಂಶವಿದೆ:

- ಓಪನ್ ಫುಡ್ ಫ್ಯಾಕ್ಟ್ಸ್ ಡೇಟಾಬೇಸ್ ಓಪನ್ ಡೇಟಾಬೇಸ್ ಲೈಸೆನ್ಸ್ (ODbL) ಅಡಿಯಲ್ಲಿ ಓಪನ್ ಡೇಟಾ ಆಗಿ ಲಭ್ಯವಿದೆ, ಕಾನೂನು ವಿವರಗಳಿಗಾಗಿ [https://world.openfoodfacts.org/terms-of-use](https://world.openfoodfacts.org/terms-of-use) ನೋಡಿ. ಎರಡು ಷರತ್ತುಗಳು ಗುಣಲಕ್ಷಣ ಮತ್ತು ಹಂಚಿಕೆ-ಸಮಾನವಾಗಿವೆ. ನೀವು ಓಪನ್ ಫುಡ್ ಫ್ಯಾಕ್ಟ್‌ಗಳಿಂದ ಡೇಟಾವನ್ನು ಇತರ ಡೇಟಾಬೇಸ್‌ಗಳೊಂದಿಗೆ ಸಂಯೋಜಿಸಿದರೆ, ODbL ಫಲಿತಾಂಶದ ಡೇಟಾಬೇಸ್ ಅನ್ನು ಓಪನ್ ಡೇಟಾ ಆಗಿ ಬಿಡುಗಡೆ ಮಾಡಬೇಕೆಂದು ಬಯಸುತ್ತದೆ. ಇದರರ್ಥ ನೀವು ಡೇಟಾವನ್ನು ಮರುಹಂಚಿಕೆಗೆ ಅವಕಾಶ ನೀಡುವ ಮೂಲಗಳೊಂದಿಗೆ ಮಾತ್ರ ಸಂಯೋಜಿಸಬಹುದು.

- ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನ್ ಅನ್ನು ಗುರುತಿಸಲು API ಕರೆಗಳನ್ನು ನಿರ್ವಹಿಸುವಾಗ ನೀವು **ಯಾವಾಗಲೂ** ಕಸ್ಟಮ್ ಬಳಕೆದಾರ-ಏಜೆಂಟ್ ಅನ್ನು ಬಳಸಬೇಕು.

- ಪ್ರತಿ API ಎಂಡ್‌ಪಾಯಿಂಟ್‌ಗೆ ದರ-ಮಿತಿಗಳನ್ನು ಜಾರಿಗೊಳಿಸಲಾಗಿದೆ.

## ಐತಿಹಾಸಿಕ ಡೇಟಾವನ್ನು ನಾನು ಹೇಗೆ ಪ್ರವೇಶಿಸಬಹುದು?

ಪ್ರಸ್ತುತ, ನಾವು ಐತಿಹಾಸಿಕ ಡೇಟಾ ಡಂಪ್ (JSONL, MongoDB, CSV) ಅನ್ನು ನೀಡುವುದಿಲ್ಲ.

ಆದಾಗ್ಯೂ, ಪ್ರತ್ಯೇಕ ಉತ್ಪನ್ನಗಳಿಗೆ, API ಬಳಸಿಕೊಂಡು ಅಥವಾ ಉತ್ಪನ್ನ ಪುಟದಲ್ಲಿ ಪರಿಷ್ಕರಣೆಗಳನ್ನು ಬಳಸಿಕೊಂಡು ಉತ್ಪನ್ನ ಡೇಟಾದ ಹಿಂದಿನ ಆವೃತ್ತಿಗಳನ್ನು ಪ್ರವೇಶಿಸಲು ಸಾಧ್ಯವಿದೆ.

ಪ್ರತಿ ಬಾರಿ ಉತ್ಪನ್ನವನ್ನು ನವೀಕರಿಸಿದಾಗ, ಹೊಸ ಪರಿಷ್ಕರಣೆ (1 ರಿಂದ ಪ್ರಾರಂಭವಾಗುವ ಹೆಚ್ಚುತ್ತಿರುವ ಅಂಕೆ) ರಚಿಸಲ್ಪಡುತ್ತದೆ.

ಉದಾಹರಣೆಗೆ, ಈ ಉತ್ಪನ್ನದ ಮೊದಲ ಪರಿಷ್ಕರಣೆ (=ಮೊದಲ ಉತ್ಪನ್ನ ಆವೃತ್ತಿ) ಪಡೆಯಲು, ಬಳಸಿ

[https://world.openfoodfacts.org/product/7623186089763/joghurt-baumnuss-migros?rev=1](https://world.openfoodfacts.org/product/7623186089763/joghurt-baumnuss-migros?rev=1).

ಅದೇ ರೀತಿ, rev ನಿಯತಾಂಕವನ್ನು API ನೊಂದಿಗೆ ಬಳಸಬಹುದು:
