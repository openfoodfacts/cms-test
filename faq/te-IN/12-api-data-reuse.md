## అప్‌లోడ్ చేసిన ఫోటోలకు ఏది మంచి సైజు అవుతుందనే దానిపై డాక్యుమెంటేషన్‌లో ఎక్కడైనా సిఫార్సులు ఉన్నాయా?

నెట్‌వర్క్ నెమ్మదిగా ఉంటే లేదా ఖరీదైనది అయితే అది దేశాలపై ఆధారపడి ఉండవచ్చు. 5000 పిక్సెల్స్ కంటే ఎక్కువ బరువు లేదా ఎత్తు ఏదైనా బహుశా అంత ఉపయోగకరంగా ఉండదు. మరియు మీరు ఆ నెట్‌వర్క్ నెమ్మదిగా ఉందని ఎలాగైనా గుర్తించగలిగితే, 2000 పిక్సెల్స్ ఇమేజ్ కూడా చాలా బాగుంటుంది (ఇమేజ్ లేకపోవడం కంటే ఖచ్చితంగా మంచిది!)

## బార్‌కోడ్‌లు లేని ఆహారం సంగతేంటి?

ఓపెన్ ఫుడ్ ఫ్యాక్ట్స్‌లో ప్యాక్ చేసిన ఆహారం గురించిన సమాచారం మాత్రమే ఉంటుంది. ఉత్పత్తి (ఉదాహరణకు, టమోటాలు లేదా అరటిపండ్లు) మరియు ఇతర ఆహార ఉత్పత్తుల సగటు విలువల కోసం, మీరు అధికారిక జాతీయ పోషకాహార డేటాబేస్‌లలో ఒకదాన్ని ఉపయోగించవచ్చు.

**గమనిక:** క్రింద ఉన్న జాబితాలో కొన్ని ముఖ్యమైన జాతీయ ఆహార డేటాబేస్‌లు ఉన్నాయి. జాబితాలో వేరే ఏదైనా డేటాబేస్ చేర్చబడాలని మీరు అనుకుంటే, దయచేసి మమ్మల్ని ఇక్కడ సంప్రదించండి: [https://world.openfoodfacts.org/contact](https://world.openfoodfacts.org/contact)

## నేను API తో ఖచ్చితమైన ఉత్పత్తి పేరును శోధించవచ్చా?

దురదృష్టవశాత్తు ఉత్పత్తి పేరు మీద మాత్రమే సులభంగా మరియు ఖచ్చితంగా API ద్వారా శోధించడం ఇంకా సాధ్యం కాలేదు.

వర్గంపై ఫిల్టర్‌ని ఉపయోగించడం వల్ల మీ శోధనను మరింత ఖచ్చితమైనదిగా చేయవచ్చు.

---

## నా ప్రాజెక్టులకు సంబంధించిన డేటాను నేను ఎలా యాక్సెస్/సేకరించగలను?

ఓపెన్ ఫుడ్ ఫ్యాక్ట్స్ ప్రధాన పేజీలో, స్క్రీన్ ఎగువ ఎడమ మూలలో, స్క్రోలింగ్ మెనూ ఉంది. దాని దిగువన, మీరు “అధునాతన శోధన” ఎంపికను కనుగొంటారు, దానిపై మీరు క్లిక్ చేయవచ్చు. మీ ప్రాజెక్ట్(లు) కు ఏ ప్రమాణాలు అత్యంత సందర్భోచితమైనవో నిర్ణయించడం మీ ఇష్టం. ఎంచుకున్న తర్వాత, మీరు పేజీ దిగువన స్క్రోల్ చేసి “ఫలితాలను డౌన్‌లోడ్ చేయి” పై క్లిక్ చేయడం ద్వారా పొందిన ఫలితాలను డౌన్‌లోడ్ చేసుకోగలరు.

మీరు వీటిని కూడా సంప్రదించవచ్చు:

- మా API డాక్యుమెంటేషన్:[ https://openfoodfacts.github.io/api-documentation/](https://openfoodfacts.github.io/api-documentation/)
- ఓపెన్ ఫుడ్ ఫ్యాక్ట్స్ యొక్క ఉపయోగ నిబంధనలు:[ ](>https://world.openfoodfacts.org/terms-of-use)[https://world.openfoodfacts.org/terms-of-use](https://world.openfoodfacts.org/terms-of-use)
- మా డేటాపై:[ https://world.openfoodfacts.org/data](https://world.openfoodfacts.org/data)

## API ని ఉపయోగించడానికి ఏవైనా షరతులు ఉన్నాయా?

API వినియోగం గురించి అన్ని డాక్యుమెంటేషన్‌లను [API డాక్యుమెంటేషన్ పేజీ](https://openfoodfacts.github.io/openfoodfacts-server/api/)లో చూడవచ్చు, కానీ ఇక్కడ ఒక చిన్న సారాంశం ఉంది:

- ఓపెన్ ఫుడ్ ఫ్యాక్ట్స్ డేటాబేస్ ఓపెన్ డేటాబేస్ లైసెన్స్ (ODbL) కింద ఓపెన్ డేటాగా అందుబాటులో ఉంది, చట్టపరమైన వివరాల కోసం [https://world.openfoodfacts.org/terms-of-use](https://world.openfoodfacts.org/terms-of-use) చూడండి. రెండు షరతులు ఆపాదింపు మరియు వాటా-ఒకే విధంగా ఉంటాయి. మీరు ఓపెన్ ఫుడ్ ఫ్యాక్ట్స్ నుండి డేటాను ఇతర డేటాబేస్‌లతో కలిపితే, ఫలిత డేటాబేస్‌ను ఓపెన్ డేటాగా విడుదల చేయాలని ODbL కోరుతుంది. దీని అర్థం మీరు డేటాను అటువంటి పునఃపంపిణీని అనుమతించే మూలాలతో మాత్రమే కలపవచ్చు.

- మీ యాప్‌ను గుర్తించడానికి API కాల్‌లను చేస్తున్నప్పుడు మీరు **ఎల్లప్పుడూ** కస్టమ్ యూజర్-ఏజెంట్‌ను ఉపయోగించాలి.

- ప్రతి API ఎండ్ పాయింట్ కు రేటు-పరిమితులు అమలు చేయబడతాయి.

## నేను చారిత్రక డేటాను ఎలా యాక్సెస్ చేయగలను?

ప్రస్తుతం, మేము చారిత్రక డేటా డంప్ (JSONL, MongoDB, CSV) ను అందించడం లేదు.

అయితే, వ్యక్తిగత ఉత్పత్తుల కోసం, APIని ఉపయోగించి లేదా ఉత్పత్తి పేజీలో సవరణలను ఉపయోగించి ఉత్పత్తి డేటా యొక్క మునుపటి సంస్కరణలను యాక్సెస్ చేయడం సాధ్యపడుతుంది.

ఒక ఉత్పత్తిని నవీకరించిన ప్రతిసారీ, ఒక కొత్త సవరణ (1 నుండి ప్రారంభమయ్యే పెరుగుతున్న అంకె) సృష్టించబడుతుంది.

ఉదాహరణకు, ఈ ఉత్పత్తి యొక్క మొదటి పునర్విమర్శ (=మొదటి ఉత్పత్తి వెర్షన్) పొందడానికి, ఉపయోగించండి

[https://world.openfoodfacts.org/product/7623186089763/joghurt-baumnuss-migros?rev=1](https://world.openfoodfacts.org/product/7623186089763/joghurt-baumnuss-migros?rev=1).

అదేవిధంగా, rev పరామితిని API తో ఉపయోగించవచ్చు:

[https://world.openfoodfacts.org/api/v2/product/7623186089763?rev=1](https://world.openfoodfacts.org/api/v2/product/7623186089763?rev=1)
