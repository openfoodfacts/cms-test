## آیا در هیچ کجای مستندات توصیه‌ای در مورد اندازه مناسب برای عکس‌های آپلود شده وجود دارد؟

این ممکن است به کشورها بستگی داشته باشد، اگر شبکه کند یا گران باشد. هر چیزی که وزن یا ارتفاع آن بالای ۵۰۰۰ پیکسل باشد، احتمالاً خیلی مفید نخواهد بود. و اگر بتوانید به نحوی تشخیص دهید که شبکه کند است، حتی یک تصویر ۲۰۰۰ پیکسلی هم عالی خواهد بود (مطمئناً بهتر از نداشتن تصویر است!)

## در مورد مواد غذایی بدون بارکد چطور؟

«اطلاعات عمومی مواد غذایی» فقط شامل اطلاعاتی در مورد مواد غذایی بسته‌بندی شده است. برای مقادیر میانگین محصولات (مثلاً گوجه فرنگی یا موز) و سایر محصولات غذایی، می‌توانید از یکی از پایگاه‌های داده رسمی تغذیه ملی استفاده کنید.

**توجه:** فهرست زیر شامل برخی از مهم‌ترین پایگاه‌های داده ملی مواد غذایی است. اگر فکر می‌کنید پایگاه داده‌ی دیگری باید در این فهرست گنجانده شود، لطفاً با ما از طریق [https://world.openfoodfacts.org/contact](https://world.openfoodfacts.org/contact) تماس بگیرید.

## آیا می‌توانم نام دقیق محصول را با API جستجو کنم؟

متأسفانه هنوز امکان جستجوی آسان و دقیق فقط بر اساس نام محصول از طریق API وجود ندارد.

استفاده از فیلتر روی دسته‌بندی می‌تواند به شما کمک کند جستجوی خود را دقیق‌تر کنید.

## چگونه می‌توانم برای پروژه‌هایم به داده‌ها دسترسی داشته باشم/داده‌ها را جمع‌آوری کنم؟

در صفحه اصلی Open Food Facts، در گوشه سمت چپ بالای صفحه، یک منوی پیمایش وجود دارد. در پایین آن، گزینه «جستجوی پیشرفته» را پیدا خواهید کرد که می‌توانید روی آن کلیک کنید. سپس این به شما بستگی دارد که تعیین کنید کدام معیارها بیشترین ارتباط را با پروژه(های) شما دارند. پس از انتخاب، می‌توانید نتایج به‌دست‌آمده را با پیمایش به پایین صفحه و کلیک بر روی «دانلود نتایج» دانلود کنید.

همچنین می‌توانید مشورت کنید:

- مستندات API ما:[ https://openfoodfacts.github.io/api-documentation/](https://openfoodfacts.github.io/api-documentation/)
- شرایط استفاده از سایت Open Food Facts:[ ](https://world.openfoodfacts.org/terms-of-use)[https://world.openfoodfacts.org/terms-of-use](https://world.openfoodfacts.org/terms-of-use)
- بر اساس داده‌های ما: [ https://world.openfoodfacts.org/data](https://world.openfoodfacts.org/data)

## آیا شرایطی برای استفاده از API وجود دارد؟

تمام مستندات مربوط به استفاده از API را می‌توان در [صفحه مستندات API](https://openfoodfacts.github.io/openfoodfacts-server/api/یافت)، اما در اینجا خلاصه‌ای سریع آمده است:

- پایگاه داده حقایق غذایی آزاد به عنوان داده باز تحت مجوز پایگاه داده باز (ODbL) در دسترس است، برای جزئیات قانونی به [https://world.openfoodfacts.org/terms-of-use](https://world.openfoodfacts.org/terms-of-use) مراجعه کنید. این دو شرط عبارتند از انتساب و اشتراک. اگر داده‌های Open Food Facts را با سایر پایگاه‌های داده ترکیب کنید، ODbL الزام می‌کند که پایگاه داده حاصل نیز به عنوان داده باز منتشر شود. همچنین به این معنی است که شما می‌توانید داده‌ها را فقط با منابعی ترکیب کنید که امکان چنین توزیع مجددی را فراهم می‌کنند.

- شما **همیشه** باید هنگام انجام فراخوانی‌های API برای شناسایی برنامه خود، از یک User-Agent سفارشی استفاده کنید.

- محدودیت‌های نرخ برای هر نقطه پایانی API اعمال می‌شود.

## چگونه می‌توانم به داده‌های تاریخی دسترسی پیدا کنم؟

در حال حاضر، ما امکان استخراج داده‌های تاریخی (JSONL، MongoDB، CSV) را ارائه نمی‌دهیم.

با این حال، برای محصولات تکی، دسترسی به نسخه‌های قبلی داده‌های محصول با استفاده از API یا در صفحه محصول با استفاده از اصلاحات امکان‌پذیر است.

هر بار که یک محصول به‌روزرسانی می‌شود، یک نسخه جدید (رقم صعودی با شروع از ۱) ایجاد می‌شود.

برای مثال، برای دریافت اولین نسخه (=اولین نسخه محصول) از این محصول، از دستور زیر استفاده کنید

[https://world.openfoodfacts.org/product/7623186089763/joghurt-baumnuss-migros?rev=1](https://world.openfoodfacts.org/product/7623186089763/joghurt-baumnuss-migros?rev=1).

به طور مشابه، پارامتر rev را می‌توان با API استفاده کرد:

[https://world.openfoodfacts.org/api/v2/product/7623186089763?rev=1](https://world.openfoodfacts.org/api/v2/product/7623186089763?rev=1)
