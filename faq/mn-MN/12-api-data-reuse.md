## Байршуулсан зургуудад ямар хэмжээтэй байх талаар баримт бичигт зөвлөмж байгаа юу?

Энэ нь сүлжээ удаан эсвэл үнэтэй эсэхээс хамаарч улс орнуудаас хамаарч болно. 5000 пикселээс дээш жин эсвэл өндөр нь тийм ч ашигтай биш байх магадлалтай. Хэрэв та сүлжээ удаан байгааг ямар нэгэн байдлаар илрүүлж чадвал 2000 пикселийн зураг ч гэсэн гайхалтай байх болно (зураггүй байснаас мэдээж дээр!)

## Баркодгүй хоолны талаар юу хэлэх вэ?

Нээлттэй хүнсний баримтууд нь зөвхөн савласан хүнсний талаарх мэдээллийг агуулдаг. Хүнсний ногоо (жишээлбэл, улаан лооль эсвэл гадил) болон бусад хүнсний бүтээгдэхүүний дундаж үнийн хувьд та оронд нь албан ёсны үндэсний хоол тэжээлийн мэдээллийн санг ашиглаж болно.

**Тэмдэглэл:** Доорх жагсаалтад үндэсний хүнсний мэдээллийн сангийн хамгийн чухал хэсгүүд багтсан болно. Хэрэв та жагсаалтад өөр мэдээллийн санг оруулах шаардлагатай гэж үзэж байгаа бол бидэнтэй дараах хаягаар холбогдоно уу: [https://world.openfoodfacts.org/contact](https://world.openfoodfacts.org/contact)

## API ашиглан бүтээгдэхүүний тодорхой нэрийг хайж болох уу?

Харамсалтай нь зөвхөн бүтээгдэхүүний нэрээр болон API-ээр дамжуулан хялбархан хайх боломжгүй байна.

Гэхдээ ангилалд шүүлтүүр ашиглах нь хайлтаа илүү нарийвчлалтай болгоход тусална.

---

## Би төслүүдийнхээ өгөгдөлд хэрхэн хандах/цуглуулах вэ?

Нээлттэй хүнсний мэдээллийн үндсэн хуудсан дээр, дэлгэцийн зүүн дээд буланд гүйлгэх цэс байдаг. Доод талд нь та "дэвшилтэт хайлт" сонголтыг олох бөгөөд та үүнийг дарж болно. Дараа нь таны төсөлд аль шалгуур хамгийн хамааралтай болохыг тодорхойлох нь танаас хамаарна. Сонгосны дараа та хуудасны доод хэсэгт доош гүйлгээд "Үр дүнг татаж авах" дээр дарж олж авсан үр дүнг татаж авах боломжтой.

Та мөн дараахтай зөвлөлдөж болно:

- Манай API баримт бичиг:[ https://openfoodfacts.github.io/api-documentation/](https://openfoodfacts.github.io/api-documentation/)
- Нээлттэй хүнсний баримтуудын хэрэглээний нөхцөл:[ ](https://world.openfoodfacts.org/terms-of-use)[https://world.openfoodfacts.org/terms-of-use](https://world.openfoodfacts.org/terms-of-use)
- Манай өгөгдөл дээр:[ https://world.openfoodfacts.org/data](https://world.openfoodfacts.org/data)

## API ашиглах нөхцөл байгаа юу?

API хэрэглээний талаарх бүх баримт бичгийг [API баримт бичгийн хуудас](https://openfoodfacts.github.io/openfoodfacts-server/api/) дээрээс олж болно, гэхдээ энд товч тойм байна:

- Нээлттэй Хүнсний Мэдээллийн Сан нь Нээлттэй Мэдээллийн Сангийн Лиценз (ODbL)-ийн дагуу нээлттэй өгөгдөл хэлбэрээр авах боломжтой бөгөөд хууль эрх зүйн дэлгэрэнгүй мэдээллийг [https://world.openfoodfacts.org/terms-of-use[https://world.openfoodfacts.org/terms-of-use] хаягаас үзнэ үү. Хоёр нөхцөл нь хамаарал болон адил тэгш хуваалцах явдал юм. Хэрэв та Open Food Facts-аас авсан өгөгдлийг бусад мэдээллийн сантай нэгтгэвэл ODbL нь үүссэн мэдээллийн санг нээлттэй өгөгдөл болгон гаргахыг шаарддаг. Энэ нь та өгөгдлийг зөвхөн ийм дахин хуваарилалтыг зөвшөөрөх эх сурвалжуудтай нэгтгэж болно гэсэн үг юм.

- Та аппликейшнаа тодорхойлохын тулд API дуудлага хийхдээ өөрчлөн тохируулсан Хэрэглэгчийн Агентыг **үргэлж** ашиглах ёстой.

- API төгсгөлийн цэг бүрт ханшийн хязгаарыг мөрддөг.

## Түүхийн мэдээлэлд хэрхэн хандах вэ?

Одоогоор бид түүхэн өгөгдлийн санг (JSONL, MongoDB, CSV) хадгалах үйлчилгээ санал болгодоггүй.

Гэсэн хэдий ч, бүтээгдэхүүн тус бүрийн хувьд бүтээгдэхүүний өгөгдлийн өмнөх хувилбаруудад API ашиглан эсвэл бүтээгдэхүүний хуудсан дээр засваруудыг ашиглан хандах боломжтой.

Бүтээгдэхүүнийг шинэчлэх бүрт шинэ хувилбар (1-ээс эхлэн цифр нэмэгдэх) үүсгэгддэг.

Жишээлбэл, энэ бүтээгдэхүүний анхны хувилбарыг (= бүтээгдэхүүний анхны хувилбар) авахын тулд ашиглана уу

[https://world.openfoodfacts.org/product/7623186089763/joghurt-baumnuss-migros?rev=1](https://world.openfoodfacts.org/product/7623186089763/joghurt-baumnuss-migros?rev=1).

Үүнтэй адилаар rev параметрийг API-тай хамт ашиглаж болно:

[https://world.openfoodfacts.org/api/v2/product/7623186089763?rev=1](https://world.openfoodfacts.org/api/v2/product/7623186089763?rev=1)
