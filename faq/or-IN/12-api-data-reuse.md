## ଅପଲୋଡ୍ ହୋଇଥିବା ଫଟୋ ପାଇଁ କେଉଁ ଆକାର ଭଲ ହେବ ସେ ବିଷୟରେ ଡକ୍ୟୁମେଣ୍ଟେସନ୍‌ରେ କୌଣସି ସ୍ଥାନରେ ସୁପାରିଶ ଅଛି କି?

ଏହା ଦେଶଗୁଡ଼ିକ ଉପରେ ନିର୍ଭର କରିପାରେ, ଯଦି ନେଟୱାର୍କ ଧୀର କିମ୍ବା ମହଙ୍ଗା। 5000 ପିକ୍ସେଲରୁ ଅଧିକ ଓଜନ କିମ୍ବା ଉଚ୍ଚତା ବୋଧହୁଏ ବହୁତ ଉପଯୋଗୀ ନୁହେଁ। ଏବଂ ଯଦି ଆପଣ କୌଣସି ପ୍ରକାରେ ଜାଣିପାରିବେ ଯେ ନେଟୱାର୍କ ଧୀର ଅଛି, ତେବେ 2000 ପିକ୍ସେଲର ଏକ ପ୍ରତିଛବି ମଧ୍ୟ ବହୁତ ଭଲ ହେବ (ପ୍ରତିଛବି ନଥିବା ଅପେକ୍ଷା ନିଶ୍ଚିତ ଭାବରେ ଭଲ!)

## ବାରକୋଡ୍ ବିନା ଖାଦ୍ୟ ବିଷୟରେ କ'ଣ?

ଖୋଲା ଖାଦ୍ୟ ତଥ୍ୟରେ କେବଳ ପ୍ୟାକେଜ୍ ହୋଇଥିବା ଖାଦ୍ୟ ବିଷୟରେ ସୂଚନା ଥାଏ। ଉତ୍ପାଦିତ ଦ୍ରବ୍ୟର ହାରାହାରି ମୂଲ୍ୟ (ଉଦାହରଣ ସ୍ୱରୂପ, ଟମାଟୋ କିମ୍ବା କଦଳୀ) ଏବଂ ଅନ୍ୟାନ୍ୟ ଖାଦ୍ୟ ଉତ୍ପାଦ ପାଇଁ, ଆପଣ ଏହା ବଦଳରେ ସରକାରୀ ଜାତୀୟ ପୁଷ୍ଟିସାର ଡାଟାବେସ୍ ମଧ୍ୟରୁ ଗୋଟିଏ ବ୍ୟବହାର କରିପାରିବେ।

**ଟିପ୍ପଣୀ:** ତଳେ ଥିବା ତାଲିକାରେ କିଛି ଗୁରୁତ୍ୱପୂର୍ଣ୍ଣ ଜାତୀୟ ଖାଦ୍ୟ ଡାଟାବେସ୍ ଅଛି। ଯଦି ଆପଣ ଭାବୁଛନ୍ତି ଯେ ତାଲିକାରେ ଅନ୍ୟ କୌଣସି ଡାଟାବେସ୍ ଅନ୍ତର୍ଭୁକ୍ତ କରାଯିବା ଉଚିତ, ଦୟାକରି ଆମ ସହିତ ଯୋଗାଯୋଗ କରନ୍ତୁ: [https://world.openfoodfacts.org/contact](https://world.openfoodfacts.org/contact)

## ମୁଁ କ’ଣ API ସାହାଯ୍ୟରେ ଏକ ସଠିକ୍ ଉତ୍ପାଦ ନାମ ଖୋଜିପାରିବି?

ଦୁର୍ଭାଗ୍ୟବଶତଃ API ମାଧ୍ୟମରେ କେବଳ ଏବଂ ସଠିକ୍ ଭାବରେ ଉତ୍ପାଦ ନାମରେ ସହଜରେ ସନ୍ଧାନ କରିବା ଏପର୍ଯ୍ୟନ୍ତ ସମ୍ଭବ ହୋଇନାହିଁ।

ବର୍ଗରେ ଫିଲ୍ଟର ବ୍ୟବହାର କରିବା ଦ୍ଵାରା ଆପଣଙ୍କ ସନ୍ଧାନକୁ ଅଧିକ ସଠିକ୍ କରିବାରେ ସାହାଯ୍ୟ ହୋଇପାରେ।

## ମୁଁ ମୋର ପ୍ରୋଜେକ୍ଟ ପାଇଁ ତଥ୍ୟ କିପରି ପ୍ରବେଶ/ସଂଗ୍ରହ କରିପାରିବି?

"ଓପନ୍ ଫୁଡ୍ ଫ୍ୟାକ୍ଟସ୍"ର ମୁଖ୍ୟ ପୃଷ୍ଠାରେ, ସ୍କ୍ରିନର ଉପର ବାମ କୋଣରେ, ଏକ ସ୍କ୍ରୋଲିଂ ମେନୁ ଅଛି। ଏହାର ତଳେ, ଆପଣ "ଉନ୍ନତ ସନ୍ଧାନ" ବିକଳ୍ପ ପାଇବେ, ଯାହା ଉପରେ ଆପଣ କ୍ଲିକ୍ କରିପାରିବେ। ତା'ପରେ ଆପଣଙ୍କ ପ୍ରକଳ୍ପ (ଗୁଡ଼ିକ) ପାଇଁ କେଉଁ ମାନଦଣ୍ଡ ସବୁଠାରୁ ପ୍ରାସଙ୍ଗିକ ତାହା ନିର୍ଣ୍ଣୟ କରିବା ଆପଣଙ୍କ ଉପରେ ନିର୍ଭର କରେ। ବାଛିବା ପରେ, ଆପଣ ପୃଷ୍ଠାର ତଳ ଭାଗରେ ସ୍କ୍ରୋଲ୍ କରି ଏବଂ “ପରିଣାମ ଡାଉନଲୋଡ୍ କରନ୍ତୁ” ଉପରେ କ୍ଲିକ୍ କରି ପ୍ରାପ୍ତ ଫଳାଫଳଗୁଡ଼ିକୁ ଡାଉନଲୋଡ୍ କରିପାରିବେ।

ଆପଣ ଏହା ମଧ୍ୟ ପରାମର୍ଶ କରିପାରିବେ:

- ଆମର API ଡକ୍ୟୁମେଣ୍ଟେସନ୍: [ https://openfoodfacts.github.io/api-documentation/](https://openfoodfacts.github.io/api-documentation/)
- ଖୋଲା ଖାଦ୍ୟ ତଥ୍ୟର ବ୍ୟବହାର ସର୍ତ୍ତାବଳୀ:[ ](https://world.openfoodfacts.org/terms-of-use)[https://world.openfoodfacts.org/terms-of-use](https://world.openfoodfacts.org/terms-of-use)
- ଆମ ତଥ୍ୟ ଉପରେ:[ https://world.openfoodfacts.org/data](https://world.openfoodfacts.org/data)

## API ବ୍ୟବହାର କରିବା ପାଇଁ କିଛି ସର୍ତ୍ତ ଅଛି କି?

API ବ୍ୟବହାର ବିଷୟରେ ସମସ୍ତ ଡକ୍ୟୁମେଣ୍ଟେସନ୍ [API ଡକ୍ୟୁମେଣ୍ଟେସନ୍ ପୃଷ୍ଠା](https://openfoodfacts.github.io/openfoodfacts-server/api/) ରେ ମିଳିପାରିବ, କିନ୍ତୁ ଏଠାରେ ଏକ ସଂକ୍ଷିପ୍ତ ସାରାଂଶ ଅଛି:

- ଓପନ୍ ଫୁଡ୍ ଫ୍ୟାକ୍ଟସ୍ ଡାଟାବେସ୍ ଓପନ୍ ଡାଟାବେସ୍ ଲାଇସେନ୍ସ (ODbL) ଅଧୀନରେ ଖୋଲା ତଥ୍ୟ ଭାବରେ ଉପଲବ୍ଧ, ଆଇନଗତ ବିବରଣୀ ପାଇଁ [https://world.openfoodfacts.org/terms-of-use](https://world.openfoodfacts.org/terms-of-use) ଦେଖନ୍ତୁ। ଦୁଇଟି ସର୍ତ୍ତ ହେଉଛି ଆଟ୍ରିବ୍ୟୁସନ୍ ଏବଂ ସେୟାର-ଆଲାଇକ୍। ଯଦି ଆପଣ Open Food Facts ରୁ ତଥ୍ୟକୁ ଅନ୍ୟ ଡାଟାବେସ୍ ସହିତ ମିଶ୍ରଣ କରନ୍ତି, ତେବେ ODbL ଆବଶ୍ୟକ କରେ ଯେ ଫଳାଫଳ ଡାଟାବେସ୍ କୁ ମଧ୍ୟ ଖୋଲା ତଥ୍ୟ ଭାବରେ ପ୍ରକାଶିତ କରାଯିବା ଆବଶ୍ୟକ। ଏହାର ଅର୍ଥ ହେଉଛି ଆପଣ କେବଳ ସେହି ଉତ୍ସଗୁଡ଼ିକ ସହିତ ତଥ୍ୟକୁ ମିଶ୍ରଣ କରିପାରିବେ ଯାହା ଏପରି ପୁନଃବଣ୍ଟନକୁ ଅନୁମତି ଦେବ।

- ଆପଣଙ୍କର ଆପ୍ ଚିହ୍ନଟ କରିବା ପାଇଁ API କଲ୍ କରିବା ସମୟରେ ଆପଣଙ୍କୁ **ସର୍ବଦା** ଏକ କଷ୍ଟମ୍ ୟୁଜର୍-ଏଜେଣ୍ଟ ବ୍ୟବହାର କରିବାକୁ ପଡିବ।

- ପ୍ରତ୍ୟେକ API ଏଣ୍ଡପଏଣ୍ଟ ପାଇଁ ହାର-ସୀମା ଲାଗୁ କରାଯାଏ।

## ମୁଁ ଐତିହାସିକ ତଥ୍ୟ କିପରି ପ୍ରବେଶ କରିପାରିବି?

ବର୍ତ୍ତମାନ, ଆମେ ଐତିହାସିକ ଡାଟା ଡମ୍ପ (JSONL, MongoDB, CSV) ପ୍ରଦାନ କରୁନାହୁଁ।

ତଥାପି, ବ୍ୟକ୍ତିଗତ ଉତ୍ପାଦଗୁଡ଼ିକ ପାଇଁ, API ବ୍ୟବହାର କରି କିମ୍ବା ସଂଶୋଧନ ବ୍ୟବହାର କରି ଉତ୍ପାଦ ପୃଷ୍ଠାରେ ଉତ୍ପାଦ ଡାଟାର ପୂର୍ବ ସଂସ୍କରଣଗୁଡ଼ିକୁ ପ୍ରବେଶ କରିବା ସମ୍ଭବ।

ପ୍ରତ୍ୟେକ ଥର ଏକ ଉତ୍ପାଦ ଅପଡେଟ୍ ହେବା ସମୟରେ, ଏକ ନୂତନ ସଂଶୋଧନ (୧ ରୁ ଆରମ୍ଭ ହେଉଥିବା ବର୍ଦ୍ଧିତ ଅଙ୍କ) ସୃଷ୍ଟି ହୁଏ।

ଉଦାହରଣ ସ୍ୱରୂପ, ଏହି ଉତ୍ପାଦର ପ୍ରଥମ ସଂଶୋଧନ (=ପ୍ରଥମ ଉତ୍ପାଦ ସଂସ୍କରଣ) ପାଇବାକୁ, ବ୍ୟବହାର କରନ୍ତୁ

[https://world.openfoodfacts.org/product/7623186089763/joghurt-baumnuss-migros?rev=1](https://world.openfoodfacts.org/product/7623186089763/joghurt-baumnuss-migros?rev=1)।

ସେହିପରି, rev ପାରାମିଟରକୁ API ସହିତ ବ୍ୟବହାର କରାଯାଇପାରିବ:
