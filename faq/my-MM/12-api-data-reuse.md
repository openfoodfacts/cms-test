## အပ်လုဒ်လုပ်ထားတဲ့ ဓာတ်ပုံတွေအတွက် သင့်တော်တဲ့ အရွယ်အစားက ဘယ်လောက်ဖြစ်မလဲဆိုတာနဲ့ ပတ်သက်ပြီး စာရွက်စာတမ်းတွေမှာ အကြံပြုချက်တွေ ရှိပါသလား။

ကွန်ရက်နှေးကွေးခြင်း သို့မဟုတ် စျေးကြီးခြင်း ရှိမရှိသည် နိုင်ငံပေါ်မူတည်နိုင်သည်။ အလေးချိန် သို့မဟုတ် အမြင့် ၅၀၀၀ pixels အထက် မည်သည့်အရာမဆို အသုံးဝင်မည်မဟုတ်ပါ။ ပြီးတော့ ကွန်ရက်နှေးကွေးနေတာကို တစ်နည်းနည်းနဲ့ သိနိုင်မယ်ဆိုရင် 2000 pixels ရှိတဲ့ ပုံတစ်ပုံတောင် အရမ်းကောင်းပါတယ် (ပုံတစ်ပုံမရှိတာထက်တော့ ပိုကောင်းတာပေါ့!)

## ဘားကုဒ်မပါတဲ့ အစားအစာတွေကော။

Open Food Facts တွင် ထုပ်ပိုးထားသော အစားအစာအကြောင်း အချက်အလက်များသာ ပါဝင်သည်။ သီးနှံများ၏ ပျမ်းမျှတန်ဖိုးများ (ဥပမာ ခရမ်းချဉ်သီး သို့မဟုတ် ငှက်ပျောသီး) နှင့် အခြားအစားအစာထုတ်ကုန်များအတွက် တရားဝင်အမျိုးသားအာဟာရဒေတာဘေ့စ်များထဲမှ တစ်ခုကို သင်အသုံးပြုနိုင်ပါသည်။

## API နဲ့ တိကျတဲ့ ထုတ်ကုန်အမည်ကို ရှာဖွေလို့ရပါသလား။

ကံမကောင်းစွာပဲ ထုတ်ကုန်အမည်တစ်ခုတည်းကို အလွယ်တကူရှာဖွေရန်နှင့် API မှတစ်ဆင့် တိကျစွာရှာဖွေရန် မဖြစ်နိုင်သေးပါ။

အမျိုးအစားအလိုက် filter ကိုအသုံးပြုခြင်းဖြင့် သင်၏ရှာဖွေမှုကို ပိုမိုတိကျစေရန် ကူညီပေးနိုင်ပါသည်။

## ကျွန်ုပ်၏ ပရောဂျက်များအတွက် ဒေတာများကို မည်သို့ ဝင်ရောက်ကြည့်ရှု/စုဆောင်းနိုင်မည်နည်း။

Open Food Facts ရဲ့ အဓိက စာမျက်နှာမှာ၊ မျက်နှာပြင်ရဲ့ ဘယ်ဘက်အပေါ်ထောင့်မှာ scroll လုပ်လို့ရတဲ့ မီနူးတစ်ခု ရှိပါတယ်။ အောက်ခြေမှာ “အဆင့်မြင့်ရှာဖွေမှု” ရွေးချယ်မှုကို သင်တွေ့လိမ့်မယ်၊ အဲဒါကို နှိပ်နိုင်ပါတယ်။ ထို့နောက် မည်သည့်စံနှုန်းများသည် သင့်ပရောဂျက်(များ)နှင့် အသင့်တော်ဆုံးဖြစ်သည်ကို ဆုံးဖြတ်ရန်မှာ သင့်အပေါ်တွင် မူတည်ပါသည်။ ရွေးချယ်ပြီးသည်နှင့် စာမျက်နှာ၏အောက်ခြေတွင် scroll လုပ်၍ “ရလဒ်များကို ဒေါင်းလုဒ်လုပ်ပါ” ကို နှိပ်ခြင်းဖြင့် ရရှိလာသောရလဒ်များကို ဒေါင်းလုဒ်လုပ်နိုင်မည်ဖြစ်သည်။

သင်လည်း တိုင်ပင်နိုင်ပါသည်-

- ကျွန်ုပ်တို့၏ API စာရွက်စာတမ်း-[ https://openfoodfacts.github.io/api-documentation/](https://openfoodfacts.github.io/api-documentation/)
- Open Food Facts ၏ အသုံးပြုမှုစည်းမျဉ်းများ-[ ](https://world.openfoodfacts.org/terms-of-use)[https://world.openfoodfacts.org/terms-of-use](https://world.openfoodfacts.org/terms-of-use)
- ကျွန်ုပ်တို့၏ဒေတာတွင်-[ https://world.openfoodfacts.org/data](https://world.openfoodfacts.org/data)

## API ကိုအသုံးပြုရန် အခြေအနေများ ရှိပါသလား။

API အသုံးပြုမှုနှင့်ပတ်သက်သည့် စာရွက်စာတမ်းအားလုံးကို [API စာရွက်စာတမ်းစာမျက်နှာ](https://openfoodfacts.github.io/openfoodfacts-server/api/) တွင် ရှာတွေ့နိုင်ပါသည်၊ သို့သော် အကျဉ်းချုပ်ကို ဤနေရာတွင် ဖော်ပြထားပါသည်။

- Open Food Facts ဒေတာဘေ့စ်ကို Open Database License (ODbL) အောက်တွင် open data အဖြစ် ရရှိနိုင်ပါသည်၊ ဥပဒေဆိုင်ရာ အသေးစိတ်အချက်အလက်များအတွက် [https://world.openfoodfacts.org/terms-of-use](https://world.openfoodfacts.org/terms-of-use) ကို ကြည့်ပါ။ အခြေအနေနှစ်ခုမှာ attribution နှင့် share-alike တို့ဖြစ်သည်။ Open Food Facts မှ အချက်အလက်များကို အခြားဒေတာဘေ့စ်များနှင့် ပေါင်းစပ်ပါက ODbL သည် ရရှိလာသော ဒေတာဘေ့စ်ကို open data အဖြစ်လည်း ထုတ်ပြန်ရန် လိုအပ်သည်။ ၎င်းသည် ထိုကဲ့သို့ ပြန်လည်ဖြန့်ဝေမှုကို ခွင့်ပြုမည့် အရင်းအမြစ်များနှင့်သာ ဒေတာကို ပေါင်းစပ်နိုင်သည်ကိုလည်း ဆိုလိုသည်။

- သင့်အက်ပ်ကို ခွဲခြားသိရှိရန် API ခေါ်ဆိုမှုများ ပြုလုပ်သည့်အခါ custom User-Agent တစ်ခုကို အမြဲအသုံးပြုရမည်။

- API endpoint တစ်ခုချင်းစီအတွက် rate-limit များကို ပြဋ္ဌာန်းထားသည်။

## သမိုင်းဝင်ဒေတာတွေကို ဘယ်လိုဝင်ရောက်ကြည့်ရှုနိုင်မလဲ။

လက်ရှိတွင်၊ ကျွန်ုပ်တို့သည် သမိုင်းဝင်ဒေတာ dump (JSONL၊ MongoDB၊ CSV) ကို မပေးဆောင်ပါ။

သို့သော်၊ တစ်ဦးချင်းထုတ်ကုန်များအတွက်၊ API ကို အသုံးပြု၍ သို့မဟုတ် ထုတ်ကုန်စာမျက်နှာတွင် ပြင်ဆင်မှုများကို အသုံးပြု၍ ထုတ်ကုန်ဒေတာ၏ ယခင်ဗားရှင်းများကို ဝင်ရောက်ကြည့်ရှုနိုင်ပါသည်။

ထုတ်ကုန်တစ်ခုကို အပ်ဒိတ်လုပ်တိုင်း၊ ပြင်ဆင်မှုအသစ် (၁ မှစတင်၍ ဂဏန်းများတိုးလာခြင်း) ကို ဖန်တီးပါသည်။

ဥပမာအားဖြင့်၊ ဤထုတ်ကုန်၏ ပထမဆုံးပြင်ဆင်မှု (=ပထမဆုံးထုတ်ကုန်ဗားရှင်း) ကိုရယူရန်၊ အသုံးပြုပါ။

[https://world.openfoodfacts.org/product/7623186089763/joghurt-baumnuss-migros?rev=1](https://world.openfoodfacts.org/product/7623186089763/joghurt-baumnuss-migros?rev=1)။

အလားတူပင်၊ rev parameter ကို API နှင့်အတူအသုံးပြုနိုင်သည်။

[https://world.openfoodfacts.org/api/v2/product/7623186089763?rev=1](https://world.openfoodfacts.org/api/v2/product/7623186089763?rev=1)
