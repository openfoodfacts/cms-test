## ມີຄໍາແນະນໍາຢູ່ບ່ອນໃດໃນເອກະສານກ່ຽວກັບຂະໜາດທີ່ເໝາະສົມສໍາລັບຮູບພາບທີ່ອັບໂຫຼດບໍ?

ນັ້ນອາດຈະຂຶ້ນກັບແຕ່ລະປະເທດ, ຖ້າເຄືອຂ່າຍຊ້າ ຫຼື ມີລາຄາແພງ. ນ້ຳໜັກ ຫຼື ຄວາມສູງທີ່ສູງກວ່າ 5000 ພິກເຊວອາດຈະບໍ່ມີປະໂຫຍດຫຼາຍ. ແລະ ຖ້າທ່ານສາມາດກວດພົບໄດ້ວ່າເຄືອຂ່າຍຊ້າ, ເຖິງແມ່ນວ່າຮູບພາບ 2000 ພິກເຊວກໍ່ຈະດີຫຼາຍ (ແນ່ນອນວ່າດີກ່ວາບໍ່ມີຮູບພາບ!)

## ແລ້ວອາຫານທີ່ບໍ່ມີບາໂຄດລ่ะ?

Open Food Facts ມີຂໍ້ມູນກ່ຽວກັບອາຫານທີ່ຫຸ້ມຫໍ່ເທົ່ານັ້ນ. ສຳລັບຄ່າສະເລ່ຍຂອງຜະລິດຕະພັນ (ຕົວຢ່າງ, ໝາກເລັ່ນ ຫຼື ໝາກກ້ວຍ) ແລະ ຜະລິດຕະພັນອາຫານອື່ນໆ, ທ່ານສາມາດໃຊ້ຖານຂໍ້ມູນໂພຊະນາການແຫ່ງຊາດຢ່າງເປັນທາງການແທນໄດ້.

**ໝາຍເຫດ:** ບັນຊີລາຍຊື່ຂ້າງລຸ່ມນີ້ປະກອບດ້ວຍຖານຂໍ້ມູນອາຫານແຫ່ງຊາດທີ່ສຳຄັນທີ່ສຸດ. ຖ້າທ່ານຄິດວ່າຖານຂໍ້ມູນອື່ນຄວນຖືກລວມເຂົ້າໃນລາຍຊື່, ກະລຸນາຕິດຕໍ່ພວກເຮົາທີ່: [https://world.openfoodfacts.org/contact](https://world.openfoodfacts.org/contact)

## ຂ້ອຍສາມາດຄົ້ນຫາຊື່ຜະລິດຕະພັນທີ່ແນ່ນອນດ້ວຍ API ໄດ້ບໍ?

ແຕ່ຫນ້າເສຍດາຍທີ່ມັນຍັງບໍ່ສາມາດຄົ້ນຫາໄດ້ງ່າຍດ້ວຍຊື່ຜະລິດຕະພັນເທົ່ານັ້ນ ແລະ ຄົ້ນຫາຜ່ານ API ໄດ້ຢ່າງແນ່ນອນ.

ການໃຊ້ຕົວກອງໃນໝວດໝູ່ອາດຊ່ວຍໃຫ້ການຄົ້ນຫາຂອງທ່ານມີຄວາມແນ່ນອນຫຼາຍຂຶ້ນໄດ້.

## ຂ້ອຍສາມາດເຂົ້າເຖິງ/ເກັບກຳຂໍ້ມູນສຳລັບໂຄງການຂອງຂ້ອຍໄດ້ແນວໃດ?

ໃນໜ້າຫຼັກຂອງ Open Food Facts, ຢູ່ມຸມຊ້າຍເທິງຂອງໜ້າຈໍ, ມີເມນູເລື່ອນໄດ້. ຢູ່ທາງລຸ່ມຂອງມັນ, ທ່ານຈະພົບເຫັນຕົວເລືອກ "ການຄົ້ນຫາຂັ້ນສູງ", ເຊິ່ງທ່ານສາມາດຄລິກໃສ່ໄດ້. ຫຼັງຈາກນັ້ນ, ມັນຂຶ້ນກັບທ່ານທີ່ຈະກໍານົດວ່າເງື່ອນໄຂໃດທີ່ກ່ຽວຂ້ອງກັບໂຄງການຂອງທ່ານຫຼາຍທີ່ສຸດ. ເມື່ອເລືອກແລ້ວ, ທ່ານຈະສາມາດດາວໂຫຼດຜົນໄດ້ຮັບທີ່ໄດ້ມາໂດຍການເລື່ອນລົງຢູ່ທາງລຸ່ມຂອງໜ້າ ແລະ ຄລິກໃສ່ “ດາວໂຫຼດຜົນໄດ້ຮັບ”.

ທ່ານຍັງສາມາດປຶກສາ:

- ເອກະສານ API ຂອງພວກເຮົາ:[ https://openfoodfacts.github.io/api-documentation/](https://openfoodfacts.github.io/api-documentation/)
- ເງື່ອນໄຂການນຳໃຊ້ຂອງ Open Food Facts:[ ](https://world.openfoodfacts.org/terms-of-use)[https://world.openfoodfacts.org/terms-of-use](https://world.openfoodfacts.org/terms-of-use)
- ກ່ຽວກັບຂໍ້ມູນຂອງພວກເຮົາ:[ https://world.openfoodfacts.org/data](https://world.openfoodfacts.org/data)

## ມີເງື່ອນໄຂໃນການໃຊ້ API ບໍ?

ເອກະສານທັງໝົດກ່ຽວກັບການໃຊ້ API ສາມາດພົບໄດ້ໃນ [ໜ້າເອກະສານ API](https://openfoodfacts.github.io/openfoodfacts-server/api/), ແຕ່ນີ້ແມ່ນບົດສະຫຼຸບໂດຍຫຍໍ້:

- ຖານຂໍ້ມູນ Open Food Facts ມີໃຫ້ເປັນຂໍ້ມູນເປີດພາຍໃຕ້ໃບອະນຸຍາດຖານຂໍ້ມູນເປີດ (ODbL), ເບິ່ງ [https://world.openfoodfacts.org/terms-of-use](https://world.openfoodfacts.org/terms-of-use) ສຳລັບລາຍລະອຽດທາງກົດໝາຍ. ເງື່ອນໄຂສອງຢ່າງຄື ການລະບຸທີ່ມາ ແລະ ການແບ່ງປັນຄືກັນ. ຖ້າທ່ານລວມຂໍ້ມູນຈາກ Open Food Facts ກັບຖານຂໍ້ມູນອື່ນໆ, ODbL ຮຽກຮ້ອງໃຫ້ຖານຂໍ້ມູນທີ່ໄດ້ຮັບຕ້ອງໄດ້ຮັບການປ່ອຍອອກມາເປັນຂໍ້ມູນເປີດເຊັ່ນກັນ. ມັນຍັງໝາຍຄວາມວ່າທ່ານສາມາດລວມຂໍ້ມູນກັບແຫຼ່ງຂໍ້ມູນທີ່ຈະອະນຸຍາດໃຫ້ມີການແຈກຢາຍຄືນໃໝ່ໄດ້ເທົ່ານັ້ນ.

- ທ່ານຕ້ອງໃຊ້ User-Agent ທີ່ກຳນົດເອງສະເໝີ ເມື່ອປະຕິບັດການເອີ້ນໃຊ້ API ເພື່ອລະບຸແອັບຂອງທ່ານ.

- ຂໍ້ຈຳກັດອັດຕາຖືກບັງຄັບໃຊ້ສຳລັບແຕ່ລະຈຸດສິ້ນສຸດ API.

## ຂ້ອຍສາມາດເຂົ້າເຖິງຂໍ້ມູນປະຫວັດສາດໄດ້ແນວໃດ?

ປະຈຸບັນ, ພວກເຮົາຍັງບໍ່ມີບໍລິການ dump ຂໍ້ມູນປະຫວັດສາດ (JSONL, MongoDB, CSV).

ເຖິງຢ່າງໃດກໍ່ຕາມ, ສຳລັບຜະລິດຕະພັນແຕ່ລະອັນ, ມັນສາມາດເຂົ້າເຖິງຂໍ້ມູນຜະລິດຕະພັນລຸ້ນກ່ອນໜ້ານີ້ໂດຍໃຊ້ API ຫຼື ໃນໜ້າຜະລິດຕະພັນໂດຍໃຊ້ການແກ້ໄຂ.

ທຸກໆຄັ້ງທີ່ຜະລິດຕະພັນຖືກອັບເດດ, ການແກ້ໄຂໃໝ່ (ຕົວເລກເພີ່ມຂຶ້ນເລີ່ມຈາກ 1) ຈະຖືກສ້າງຂຶ້ນ.

ຕົວຢ່າງ, ເພື່ອໃຫ້ໄດ້ການແກ້ໄຂຄັ້ງທຳອິດ (=ລຸ້ນຜະລິດຕະພັນທຳອິດ) ຂອງຜະລິດຕະພັນນີ້, ໃຫ້ໃຊ້

[https://world.openfoodfacts.org/product/7623186089763/joghurt-baumnuss-migros?rev=1](https://world.openfoodfacts.org/product/7623186089763/joghurt-baumnuss-migros?rev=1).

ເຊັ່ນດຽວກັນ, ພາລາມິເຕີ rev ສາມາດໃຊ້ກັບ API ໄດ້:
